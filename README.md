## 1.2.0 基本实现完整界面展示，后续预计只需美化。

### 1. 界面风格上的改进

之前那版更像：

- 功能验证页面
- 普通表单式布局
- 重点在“能跑通”

现在这版更偏向：

- 现代化
- 年轻化
- 玻璃拟物风（Glassmorphism）
- 更适合展示和答辩

具体变化包括：

- 增加了渐变氛围背景
- 卡片统一采用玻璃质感
- 圆角、阴影、透明度、模糊背景更统一
- 操作按钮视觉层级更清楚
- 页面整体更像完整产品，而不是开发测试页

------

### 2. 功能上的改进

之前那版主要是基础功能：

- 上传图片
- 调接口
- 显示原图
- 显示可视化结果
- 显示识别表格
- 复制/导出

现在这版新增了不少实用功能：

#### 上传相关

- 点击选择图片
- 拖拽上传
- `Ctrl + V` 粘贴图片
- 上传进度显示

#### 图像查看相关

- 原图和检测结果分开展示
- 支持缩放
- 支持滚轮缩放
- 支持复位
- 支持拖拽查看放大后的局部细节
- 默认尽量居中显示

#### 结果筛选相关

- 新增置信度阈值滑块
- 可以按阈值过滤识别结果
- 低于阈值的结果不显示

#### 结果交互相关

- 检测结果区支持显示/隐藏框中文字
- 框中文字支持鼠标划取复制
- 合并文本区会随阈值变化同步更新

#### 页面组织相关

- 底部改成标签页
- 把“识别结果 / 日志 / 模型说明 / 系统说明”分开
- 整体更紧凑，不容易把页面拉得很长

------

### 3. 交互体验上的改进

这一部分其实是提升最大的。

之前那版更多是：

- 点一下按钮
- 看结果

现在这版更强调“过程反馈”和“细节体验”：

- 识别时有加载遮罩
- 有进度条
- 按钮禁用逻辑更完善
- 空状态提示更明确
- 图片框本身就能承担上传入口
- 放大后的图片支持拖拽查看
- 结果区有缩放和阈值控制
- 不同状态下有更清晰的视觉反馈

也就是说，之前更像“会用的人能操作”，现在更像“普通用户也知道怎么用”。

------

### 4. 展示逻辑上的改进

这点特别适合写进毕业设计。

之前那版是“功能堆叠”：

- 原图
- 可视化图
- 结果表格
- 日志

都按顺序往下放。

现在这版是“信息分层展示”：

#### 第一层：核心操作

- 选择图片
- 识别
- 导出
- 阈值控制

#### 第二层：关键状态

- 当前文件
- 原始行数
- 当前显示
- 当前状态

#### 第三层：核心结果

- 原图
- 检测结果
- 缩放查看

#### 第四层：补充信息

- 识别结果表格
- 日志
- 模型说明
- 系统说明
